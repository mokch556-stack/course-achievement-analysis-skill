# 课程教学目标达成度分析技能

## 什么时候用

当用户需要：
- 为课程计算各教学目标达成度
- 生成教务处模板格式的**达成度计算表**（.xlsx），含Sheet1数据表、Sheet2映射关系、Sheet3示例学生
- 生成**附件7：课程教学目标达成情况分析报告**（.docx），含4段式结构
- 对应附件6清单要求：终结性考核典型性错误及试题难度分析、课程教学目标达成分析、存在问题、改进意见

**不要用**：
- 只需录入成绩（用成绩录入技能）
- 只需听课评价（用听课评价技能）
- 只需创建成果申报表（用 create_achievement_form 技能）

## 前置检查

| # | 检查项 | 验证方式 |
|---|--------|---------|
| 1 | 教学大纲.docx 存在（含课点归属表格） | 读取文件确认 |
| 2 | 开课说明.docx 存在（含考核构成、二级矩阵） | 读取文件确认 |
| 3 | 成绩登记表.xlsx（有平时+期末列） | 读取文件确认，**col 5=期末成绩** |
| 4 | 教务处附件模板（附件5/7）存在 | 检查模板文件 |
| 5 | 课程基本信息已知（学期/学分/考核方式/专业/班级） | 从文件提取 |

## 三步工作流

### 第一步：生成空白达成度计算表（含映射+示例）——多步骤追问用户确认！

**⚠️ 核心铁律：这一步必须分多次向用户追问确认，用户签了字才能走下一步！绝不自己猜！**

#### 追问流程（严格执行，缺一不可）：

**追问①：课程基本信息确认**
```
请确认：
- 课程名称：______
- 学期：_________
- 班级：_________
- 任课教师：______
- 学分：_________
```
→ 从教学大纲/开课说明提取后，**逐条念给用户确认**

**追问②：考核构成确认**
```
考核构成：
- 平时成绩占___%（含___个子项：______）
- 期末成绩占___%（考核方式：______）
- 总评 = 平时×___% + 期末×___%
以上是否正确？
```
→ 从开课说明提取，**念给用户确认**

**追问③：教学目标+课点归属确认**
```
教学目标（共K个）：
1. ______（__课点）
2. ______（__课点）
...
各考核子项归属于哪些目标？我初步梳理如下：
- 项目一（30分）→ 目标__
- 项目二（30分）→ 目标__
...
请确认课点归属关系
```
→ 从教学大纲表格②提取，**念给用户确认**

**追问④：期末评分标准确认**
```
期末项目作业评分维度：
- ______（__分）
- ______（__分）
- ______（__分）
- ______（__分）
以上是否准确？有无补充？
```
→ 从项目作业指导书提取，**念给用户确认**

**追问⑤：满分矩阵草稿确认**
```
经过以上数据，我算出的满分矩阵如下（Sheet2）：

考核项目 | 满分 | 目标1 | 目标2 | ... | 目标K
---------|------|-------|-------|-----|------
...      | ...  | ...   | ...   | ... | ...
合计     | 100  | xx    | xx    | xx  | xx

各目标权重是否与教学大纲一致？确认后我生成模板。
```
→ **算出矩阵后先给用户看，等确认再动手**

**追问⑥：示例学生人选确认**
```
示例学生我打算选______（姓名，分数居中偏下），
这样能展示各维度的计算全貌。是否可以？
```
→ **等用户点头**

---

**所有追问都确认后** → 生成空白模板 xlsx，含3个Sheet：

#### Sheet1：表头信息
- 顶部：课程名、学期、任课教师、学分等基本信息
- 表头行：序号 | 学号 | 姓名 | 平时总分 → 子项1~N | 期末总分 → 期末子项1~M | 总评 | 折合分 → 目标1~目标K（合计=100）
- 表头下层（合并单元格）：各考核项目→对应目标
- ✅ 数据行留空待填

#### Sheet2：映射关系表（满分分解矩阵）
| 考核项目 | 满分 | 目标1 | 目标2 | ... | 目标K | 说明 |
|----------|------|-------|-------|-----|-------|------|
| 课堂表现-项目一 | 30 | 13.44 | 0 | 0 | 0 | 按课点归属 |
| 课程竞赛 | 20 | 7.14 | 3.57 | 3.57 | 5.72 | 按课点比 |
| ... | ... | ... | ... | ... | ... | ... |
| 期末项目作业 | 100 | 40 | 10 | 20 | 30 | 按评分标准4维度 |
| **合计** | **100** | **40** | **11.5** | **19.5** | **29** | **各目标满分** |

**满分矩阵计算规则**：
- 课堂表现（阶段性测试）：按项目内容归属到对应目标
- 通识子项（学习通/课程竞赛/课堂笔记/实训报告）：按**课点比例**分配到各目标
  - 各目标课点数来自教学大纲表格②
- 期末项目作业：按评分标准4维度对应
- **笔记列需补偿调整**：因课堂表现变化，需要微调笔记列的分配使各目标合计=教学大纲规定的比例
- ⚠️ 合计横向=各目标满分（分别对应教学大纲规定的权重×100）

#### Sheet3：示例学生1个
- 复制Sheet1的一行数据，填入选定的示例学生的实际分
- 展示完整的计算链路：原始分→折合分→达成度
- 旁边加**列注释**说明每列的计算公式和数据来源
- **作用**：让用户一眼看明白数据怎么填、达成度怎么算

---

### 第二步：产出文件（填好的计算表 + 分析报告Word）

#### 2a：填写达成度计算表

1. 把Sheet1模板逐行填入所有学生数据
2. 计算每个学生各项目标的折合分和达成度
3. 按教务处模板格式排版：
   - 达成度值**四舍五入保留2位小数**（0.7613→0.76）
   - 行75按目标合并单元格（D-H、I-M、N-R、S-W各合并为一个）
   - 行76课程达成度合并
   - 数据行无底色，底部行保留底色
4. 保存为 `达成度计算表_{课程名}_{班级}.xlsx`

#### 2b：生成附件7分析报告（docx）

**⚠️ 必须使用XML层操作修改表格**，不用 `cell()` API：

```python
def find_text_run(p):
    """在段落中查找含有非空<w:t>的run（不一定在第一个run）"""
    for run in p.findall(f'{W}r'):
        t = run.find(f'{W}t')
        if t is not None and t.text and t.text.strip():
            return t
    return p.find(f'.//{W}t')
```

**报告4段式结构（对齐附件6清单）**：

| 章节 | 行 | 内容 | 数据源 |
|------|----|------|--------|
| 一、终结性考核典型性错误及试题难度分析 | R21 | ①典型错误分析（按评分标准4维度）②难度系数/区分度 ③全年级+各班的成绩分布表 | 成绩登记表col 6 |
| 二、课程教学目标达成分析 | R22 | 各目标达成度值+课程整体达成度 | 达成度计算表 |
| 三、存在问题 | R23 | 学生问题（4维度薄弱点）+ 教师教学改进方向（建设性措辞） | 成绩数据+教学观察 |
| 四、改进意见 | R24 | 具体改进措施（对应存在问题） | 综合分析 |

**关键词替换规则**（当R21描述终结性考核时）：
- ❌ "三次项目作业/项目一二三" → ✅ "期末项目作业（综合性分析报告）"
- ❌ 按题型分析 → ✅ 按评分标准4维度分析典型错误

**教师问题建设性措辞规则**（参考新媒体报告写法）：

| ❌ 指控性 | ✅ 建设性 |
|-----------|----------|
| 教师未提供... | 辅导资源可进一步丰富/完善 |
| ...不足/不够 | 可进一步加强/优化 |
| 缺乏... | 有待加强... |
| 帮扶不够 | 过程性关注和帮扶机制可进一步加强 |
| 预警不足 | 学业预警机制可进一步优化 |
| "教师存在的问题" | → "教师教学改进方向" |

**成绩分布数据写入规则**：
- R12表单：全年级各分数段人数+百分比（如90-100:8(12.50%)）
- R17模板：各班最高/最低/均分/及格率 + 学生姓名（如最高95/赵铭方，最低49/董鑫蓥）
- 不及格学生：全部列出姓名+分数（如11人：王艳丽52分，张艳萍51分...）

---

### 第三步：核验（三核查）+ 整理 + 打开

#### 3a：数据核验（三核查必做，不可跳过）

| 维度 | 检查项 | 通过条件 | 验证方法 |
|------|--------|---------|---------|
| 数据一致性 | ①成绩分布（各分数段人数） ②最高/最低分+姓名 ③平均分 ④及格率 ⑤不及格学生名单+分数 | 与成绩登记表原始数据 100% 一致 | Python脚本逐条对照 |
| 教学大纲一致性 | ①教学目标数量+名称 ②学分 ③考核方式 ④课点归属 | 与教学大纲.docx一致 | 读取文件对照 |
| 开课说明一致性 | ①学期 ②考核构成 ③成绩比例 ④二级矩阵 | 与开课说明.docx一致 | 读取文件对照 |

**⚠️ 成绩登记表列结构关键**：
- col 4 = 平时成绩，col 5 = 期末成绩（不是总评col 7！）
- 两个班列结构**完全相同**（先探查表头行再确认）
- ⚠️ 成绩分布必须用期末成绩col 5（不是总评，也不是平时）

**⚠️ 常见错误（已有血泪教训）**：
| 错误 | 后果 | 预防 |
|------|------|------|
| 误用总评成绩算分布 | 分布全错 | 必须col 5 |
| 232班列错看成不同结构 | 分布全错 | 两班分别探查 |
| 达成度显示4位小数 | 不合模板要求 | 统一2位 |
| docx模板0.00%残留 | 格式异常 | XML层set_tc_text |
| 学期号写错 | 全校性错误 | 对照开课说明签署日期 |

#### 3b：如有必要 → 打补丁

发现核验结果有差异时：
1. 定位具体哪个数据/格式有问题
2. 写**定向修复脚本**（只改错的那个位置，不动好的部分）
3. 修复后重新核验
4. 修复脚本归档到 `④脚本工具\`

#### 3c：文件夹整理

按照规范结构整理输出：

```
{output_dir}\{课程名}\
├── ①原始数据\          成绩登记表、项目指导书、原始成绩单（只保留最终版）
├── ②参考文档\          教学大纲、开课说明、核查映射表
├── ③最终成果\          仅最终交付件（达成度表.xlsx + 附件7.docx），其他一概不留
├── ④脚本工具\          Python脚本（展平为文件，无子目录）
├── ⑤附件模板\          教务处模板
└── ⑥_废弃_中间文件\    中间文件归档（不删除！便于回溯）
```

**最终成果精简原则**：
❌ 不保留：中间版本、改动说明、独立图表、重复备份、临时笔记
✅ 只保留：`达成度计算表_{课程名}_{班级}.xlsx` + `附件7_{课程名}_教学目标达成情况分析报告.docx`

#### 3d：打开给用户

```
start "" "③最终成果\附件7_{课程名}_教学目标达成情况分析报告.docx"
start "" "③最终成果\达成度计算表_{课程名}_{班级}.xlsx"
```

同时发送一条 QQ 通知（合并式，带签名）。

## 特殊场景处理

### 学期判定
- 开课说明签署日期决定学期：如2026年2月25日 → 2025-2026学年第2学期
- 日期写法：6月30日查成绩 → 年份与学期结尾一致（如2025-2026-2 → 2026年6月30日）

### 终结性考核名称
- ✅ 正确定义：**期末项目作业**（综合性分析报告），不是"三次项目作业"
- 阶段性测验（项目一二三）属于平时考核的一部分，不是终结性考核
- 期末项目作业评分标准：见项目作业指导书.doc（如维度再确认→4维度）

### 图表配置
- 报告末尾按需补充4张图：成绩分布柱状图/分班对比图/目标达成度对比图/考核环节雷达图
- 图表用matplotlib生成，dpi≥200

## 避坑速查

| 坑 | 解决方法 |
|----|---------|
| 期末成绩用了总评 | 必须col5，不是总评列 |
| 两班列结构不同 | 分别探查表头 |
| 达成度4位小数 | 保留2位 |
| docx表格格式乱 | XML层操作，不用cell() |
| 重复段落显示 | P0内部裁为单套，或清冗余段落 |
| 0.00%模板残留 | 用set_tc_text填入正确值 |
| 教师措辞尖锐 | 用建设性表述 |
| 学期号写错 | 对照开课说明签署日期 |
| 分布数据用了平时成绩 | 用期末成绩col5 |
