# 课程教学目标达成度分析技能

## 什么时候用

当用户需要：
- 为课程计算各教学目标达成度
- 生成教务处模板格式的**达成度计算表**（.xlsx）
- 生成**附件7：课程教学目标达成情况分析报告**（.docx），含4段式结构
- 对应附件6清单要求：终结性考核典型性错误及试题难度分析、课程教学目标达成分析、存在问题、改进意见

**不要用**：
- 只需录入成绩（用成绩录入技能）
- 只需听课评价（用听课评价技能）
- 只需创建成果申报表（用 create_achievement_form 技能）

## 前置检查

| # | 检查项 | 验证方式 |
|---|--------|---------|
| 1 | 教学大纲.docx 存在 | 读取文件确认 |
| 2 | 开课说明.docx 存在 | 读取文件确认 |
| 3 | 成绩登记表.xlsx（有平时+期末列）存在 | 读取文件确认 |
| 4 | 教务处附件模板（附件5/7）存在 | 检查模板文件 |
| 5 | 课程基本信息已知（学期/学分/考核方式/专业/班级） | 从教学大纲+开课说明提取 |

## 核心工作流

### 输入数据

| 数据源 | 关键信息 | 用途 |
|--------|---------|------|
| 教学大纲.docx | 课程名称、教学目标（4个目标）、课点列表、课点归属 | 构建满分行 + 公式 |
| 开课说明.docx | 考核方式、成绩构成比例、二级矩阵、学期、学分 | 验证考核结构 |
| 成绩登记表.xlsx | 学生平时成绩（学习通/笔记/竞赛/实训）+ 项目考试成绩（3次） | 计算依据 |
| 教务处模板 | 附件5（达成度计算表模板）、附件7（分析报告模板） | 格式合规 |

---

### 步骤 1：读懂课程结构

#### 1A. 提取教学大纲关键信息

```
教学目标（通常4个）：
  目标1：掌握xxx能力 → 满分40
  目标2：掌握xxx能力 → 满分11.5
  目标3：掌握xxx能力 → 满分19.5
  目标4：掌握xxx能力 → 满分29

课点归属映射（示例）：
  课点1-12 → 目标1(10个) + 目标4(2个)
  课点13-20 → 目标2(4个) + 目标4(4个)
  课点21-32 → 目标1(4个) + 目标3(6个) + 目标4(2个)
```

#### 1B. 提取开课说明考核结构

```
考核环节：
  平时成绩（40分）：
    学习通成绩 20分（线上视频/测验/课堂表现）
    课堂笔记  4分
    课程竞赛  8分
    实训报告  8分
  项目考试（60分）：3次项目作业，各25分 = 75分（折算为60分）
    项目一 → 目标1(10/12) + 目标4(2/12)
    项目二 → 目标2(4/8) + 目标4(4/8)
    项目三 → 目标1(4/12) + 目标3(6/12) + 目标4(2/12)
```

#### 1C. 构建满分矩阵（核心）

按教学目标拆解各项考核满分：

| 目标 | 学习通 | 笔记 | 竞赛 | 实训 | 项目考试 | **合计满分** |
|------|--------|------|------|------|---------|:----------:|
| 目标1 | 8.26 | 2.24 | 3.5 | 2.0 | 24.0 | **40** |
| 目标2 | 2.92 | 0.58 | 1.0 | 1.0 | 6.0 | **11.5** |
| 目标3 | 3.54 | 0.46 | 1.5 | 2.0 | 12.0 | **19.5** |
| 目标4 | 5.28 | 0.72 | 2.0 | 3.0 | 18.0 | **29** |
| **合计** | **20** | **4** | **8** | **8** | **60** | **100** |

**计算规则**：
- 学习通/笔记/竞赛/实训：按各目标下课点比例分配
- 项目考试：按课点归属比例分配到各目标
- 笔记列需补偿调整（使各目标合计=教学大纲规定比例）

---

### 步骤 2：提取成绩数据

#### 2A. 从成绩登记表提取

成绩登记表列结构（标准）：

| 列 | 内容 |
|:--:|------|
| 1 | 序号 |
| 2 | 姓名 |
| 3 | 学号 |
| 4 | 平时成绩（总分40） |
| 5 | 期末成绩（项目考试总分） |

**⚠️ 关键规则**：
- 期末成绩所在列 = col 5（不是 col 4！）
- 两个班的列结构相同
- 先通过表头确认，再提取数据

#### 2B. 逐学生计算达成度

对每个学生，按满分矩阵比例计算折合分：

```
目标1折合分 = 学习通成绩 × (8.26/20) + ... + 项目成绩 × (24.0/60)
目标达成度 = 该目标实际折合分 / 该目标满分
课程达成度 = 所有目标折合分之和 / 100（即总折合分/满分）
```

### 步骤 3：生成达成度计算表（xlsx）

#### 3A. 模板格式

使用教务处附件5模板格式：

- **行1-5**：课程信息（课程名、学分、考核方式、专业、班级等）
- **行6+**：学生数据（每行一名学生）
  - 学号、姓名、平时各项成绩、项目考试各次成绩
  - 按目标分配的折合分（目标1-4各列）
  - 总分列
- **倒数第2行**：各目标平均值 + 达成度值
- **倒数第1行**：课程整体达成度

#### 3B. 关键格式规则

| 规则 | 说明 |
|------|------|
| 达成度值 | **四舍五入保留2位小数**（0.7613→0.76） |
| 行75教学目标达成度 | 按目标合并单元格（D-H、I-M、N-R、S-W各合并为一个） |
| 行76课程达成度 | 合并所有达成度列为一个单元格 |
| 平均值 | 保留2位小数 |
| 底色 | 数据行无底色，底部平均/达成度行保留底色 |
| 数据完整性 | 确保64名（或其他人数）学生全部包含 |

#### 3C. 生成逻辑（Python）

```python
from openpyxl import Workbook, load_workbook

# 1. 按满分矩阵计算各目标折合分
# 2. 写入模板格式
# 3. 计算班均和目标达成度
# 4. 格式设置（小数位数、合并单元格）
```

### 步骤 4：生成附件7分析报告（docx）

#### 4A. 模板操作

使用教务处附件7模板（.docx），通过 **XML层操作** 修改表格内容。

**⚠️ 关键规则**：
- 不能使用 `python-docx` 的 `cell()` API 修改内容（格式会乱）
- 必须使用 `lxml` XML层操作：
  ```python
  from docx import Document
  from lxml import etree
  
  W = '{http://schemas.openxmlformats.org/wordprocessingml/2006/main}'
  
  # 定位单元格
  tc = table.rows[row_idx]._tr.findall(f'{W}tc')[col_idx]
  
  # 查找文本run（文字不一定在第一个run）
  def find_text_run(p):
      """在段落中查找含有非空<w:t>的run"""
      for run in p.findall(f'{W}r'):
          t = run.find(f'{W}t')
          if t is not None and t.text and t.text.strip():
              return t
      # 兜底：用第一个run
      first_t = p.find(f'.//{W}t')
      return first_t
  
  # 设置文本
  def set_tc_text(tc, text):
      """在TC的第一个段落中设置文本，保留格式"""
      p = tc.find(f'{W}p')
      if p is None:
          return
      # 清除多余run（只保留第一个有文字的run）
      runs = p.findall(f'{W}r')
      target_t = find_text_run(p)
      if target_t is not None:
          target_t.text = text
      # 清除其他run
      for run in runs:
          t = run.find(f'{W}t')
          if t is not None and t is not target_t:
              t.text = ''
  ```

#### 4B. 分析报告4段式结构

按教务处附件6清单要求：

| 章节 | 表格行 | 内容要求 |
|------|--------|---------|
| 一、终结性考核典型性错误及试题难度分析 | R21 | 典型错误描述 + 试题难度分析 + 成绩分布 |
| 二、课程教学目标达成分析 | R22 | 各目标达成度 + 课程整体达成度 |
| 三、存在问题 | R23 | 学生问题 + 教师教学改进方向（建设性措辞） |
| 四、改进意见 | R24 | 具体改进措施 |

#### 4C. 成绩分布表（Table0）

Table0 存放成绩分布数据，格式：

| 班级 | 90-100 | 80-89 | 70-79 | 60-69 | 50-59 | 40-49 | 40↓ |
|------|--------|-------|-------|-------|-------|-------|-----|
| 全年级 | 8 | 21 | 21 | 3 | 10 | 1 | 0 |
| 231班 | 6 | 12 | 8 | 1 | 5 | 1 | 0 |
| 232班 | 2 | 9 | 13 | 2 | 5 | 0 | 0 |

#### 4D. 教师问题建设性措辞规则

| ❌ 指控性措辞 | ✅ 建设性措辞 |
|---------------|--------------|
| 教师未提供充分的模板参考 | 辅导资源可进一步丰富 |
| 辅导与支持不足 | 辅导资源可进一步丰富 |
| 重点关注和帮扶不够 | 过程性关注和帮扶机制可进一步加强 |
| 缺乏针对性的督促 | 需建立更有针对性的督促机制 |
| 过程性监测和预警不足 | 过程性干预机制可进一步优化 |
| 区分度不够显著 | 整体均衡，可在…上进一步强化 |
| 教师存在的问题 | 教师教学改进方向 |

### 步骤 5：核查验证（必做！最后一步）

必须执行三个维度的核查：

#### 5A. 数据一致性核查

| 检查项 | 方法 | 通过条件 |
|--------|------|---------|
| 期末成绩分布 | 从成绩登记表col5逐条读取，与报告对比 | 各分数段人数一致 |
| 平时成绩 | 对比成绩登记表col4与计算表中的平时分 | 100%一致 |
| 最高/低分 | 从成绩登记表提取，与报告对比 | 姓名+分数一致 |
| 不及格名单 | 逐条核对 | 人数+姓名+分数正确 |
| 均分/及格率 | 从成绩登记表计算，与报告对比 | 误差≤0.5% |

#### 5B. 教学大纲一致性核查

| 检查项 | 方法 | 通过条件 |
|--------|------|---------|
| 教学目标 | 对比教学大纲目标描述与报告 | 一致 |
| 学分/考核方式 | 对比教学大纲 | 一致 |
| 课程名称 | 对比教学大纲 | 一致 |

#### 5C. 开课说明一致性核查

| 检查项 | 方法 | 通过条件 |
|--------|------|---------|
| 学期 | 检查报告中的学期与开课说明签署日期 | 对应正确 |
| 考核构成 | 对比开课说明的考核权重与计算表 | 一致 |
| 成绩构成比例 | 对比开课说明 | 一致 |

---

## 避坑速查

| 坑 | 解决方法 |
|----|---------|
| 期末成绩用了总评成绩 | 必须从成绩登记表col5提取，不是总评/平时列 |
| 两个班列结构不一致 | 分别探查每个表的表头和列结构 |
| 达成度显示4位小数 | 四舍五入保留2位（0.7613→0.76） |
| docx表格修改后格式乱 | 用XML层操作（set_tc_text/find_text_run），不用cell() API |
| 跨run文字（如学期） | 清除多余run的<w:t>文本，只保留目标run的文字 |
| 报告中出现重复段落 | P0内部有3套重复 → 裁为单套；或P0+多段落重复 → 清冗余段落 |
| 成绩分布表中0.00%模板残留 | 用set_tc_text填入正确值 |
| 班长姓名实际为女生名 | 需从成绩登记表核对姓名性别 |
| 教师措辞过于尖锐（如"教学事故"级别） | 改为建设性表述（"可进一步""有待加强""可优化"） |
