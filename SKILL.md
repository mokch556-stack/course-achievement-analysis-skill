# 课程教学目标达成度分析 v4

> 操作时间: 2026-06-30 15:30
> 操作agent: default
> v4 核心变更（v3→v4）:
>   - xlsx 格式改为学校标准宽表格式（参考丁向朝老师卷三3-达成度表格）
>   - 单Sheet宽表：20列折合后数据 + 1列SUM公式（折合得分≤100）
>   - zhe_full 修正：线上/手册/课堂=2.5, 阶段测试=5, 期末考试=12.5（每目标25×4=100）
>   - docx 改为从 config 直接计算达成度（不依赖 Excel 公式求值）
>   - 达成度计算：折合分 = raw/raw_full × zhe_full_per_target, 折合总分 = SUM(20列) ≤ 100
> v3 核心变更（从v2重写）:
>   - 新增 Step 0「数据归集」：从课程档案源文件自动提取所有学生分数，禁止手工编分数
>   - 子项→目标映射自动从教学大纲/开课说明提取，阶段测试按项目归属拆分
>   - 按专业（而非班级）分表生成：一个专业一张达成度计算表 + 一份附件7报告

## 🧠 自主智能规则（铁律，不可跳过）

以下规则内化为 Skill 的自动行为，**不需要用户提示**：

### 规则1：xlsx 文件必须遍历全部 Sheet
- `*平时成绩报告单*.xlsx` 等文件可能有多个 Sheet（如"模版"放A专业，"Sheet1"放B专业）
- 归集时 **MUST 遍历 `wb.sheetnames` 中的所有 Sheet**，不得只读 `wb.active`
- 跳过明显非数据Sheet（如名称为纯数字的汇总Sheet），其余全读

### 规则2：子项→目标映射必须从教学大纲/开课说明提取
- 进入课程档案父目录，搜索 `*教学大纲*.docx` 和 `*开课说明*.docx`
- 用 pandoc 提取文本，读取考核评价方式、项目→目标对应关系
- **阶段测试按项目归属**：项目1→目标1，项目2→目标2，以此类推
- **覆盖性考核（线上/课堂/手册/期末）**：均匀分配到所有目标
- 如果找不到大纲/开课说明，才问用户，但必须先尝试自动发现

### 规则3：按专业分表，非按班级
- 一个专业（major）生成一张达成度计算表 + 一份附件7报告
- 从 config 的 `major` 字段获取专业名，或从班级名提取（如"互金251"→"互联网金融"）
- 不同专业的达成度独立计算、独立报告

### 规则4：阶段测试独立保留原始分
- 阶段测试1/2/3/4 各自保留满分100的原始分，不提前折合不取平均
- 在达成度计算时各自折合到对应目标（`raw/raw_full × zhe_full`）

## 工作流（4步）

```
Step 0: 探查+归集 → 扫描目录结构 + 读教学大纲 + aggregate_scores.py
Step 1: 构建 config → 按专业拆分配置，阶段测试分目标
Step 2: 生成 xlsx  → build_xlsx_v3.py（每专业一张表）
Step 3: 生成 docx  → build_docx_v3.py（每专业一份附件7）
```

### Step 0: 探查目录 + 数据归集

#### 0a. 自动探查（强制执行）

1. 扫描课程档案目录，列出所有 `.xlsx` 文件 + 每个文件的 Sheet 名和列名
2. 自动发现文件类型：平时成绩报告单、综合成绩、手册成绩、课堂表现、阶段测试、线上学习、PDF登记表
3. **向上2级目录**搜索 `*教学大纲*.docx` 和 `*开课说明*.docx`
4. pandoc 提取文本 → 读取考核评价方式、项目→目标映射

#### 0b. 归集成绩

**脚本**：`scripts/aggregate_scores.py`

**用法**：
```python
from aggregate_scores import aggregate
students = aggregate("课程档案目录路径")
# 返回: [{xh, name, class, major, raw_scores: {子项名: 分}, luru_ps, luru_qm, luru_zp}, ...]
```

**归集策略**（脚本内置）：
1. 遍历平时成绩报告单的所有 Sheet → 提取线上/课堂/测试/手册折合分
2. 综合成绩表 → 补班级+姓名
3. 子项源文件（手册/课堂/阶段测试1-4/线上）→ 补缺
4. PDF 成绩登记表 → luru_ps/luru_qm/luru_zp
5. 线上成绩只数视频名列（C4/C6/C8…），忽略相邻状态列

**字段说明**：
- `targets[].items[].raw_full`：该考核环节原始满分
- `targets[].items[].zhe_full`：分配到该目标的折合满分
- `students[].raw_scores`：各子项得分（阶段测试为原始100分制，其余已折合）
- `luru_ps/luru_qm/luru_zp`：PDF登记表录入成绩（仅校验用）

### Step 2: 生成 xlsx（学校标准宽表格式）

```bash
python scripts/build_xlsx_school.py config_{专业名}.json output.xlsx
```

单 Sheet 学校格式（参考丁向朝老师达成度表格）：
- **R1-R3**：标题 + 专业/班级/学期 + 课程/代码/教师
- **R4-R7**：表头（序号|学号|姓名|毕业要求指标|4×5子项名|满分行|备注）
- **R8+**：学生数据行，**折合得分列 = SUM(Dn:Wn) 公式**（满分100）
- **平均分行 + 达成度行**：AVERAGE 公式 + SUM/SUM 达成度公式

折合分 per cell = raw / raw_full × zhe_full_per_target。折合总分 = SUM(20列) ≤ 100。

### Step 3: 生成 docx（从 config 直接计算）

```bash
python scripts/build_docx_v3.py config_{专业名}.json template.docx xlsx_path output.docx
```

从教务处空白模板生成 4 段式分析报告。

## 注意事项

1. **数据归集是强制步骤**：不得手工编分数，必须从源文件提取
2. **必须遍历所有 Sheet**：平时成绩报告单等文件可能分Sheet存放不同班级
3. **子项→目标映射从教学大纲/开课说明提取**：阶段测试按项目归属，覆盖性考核均匀分配
4. **按专业分表**：一个专业一张达成度表 + 一份附件7报告
5. **阶段测试保留原始100分制**：不提前折合，在达成度计算时各自折合到对应目标
6. **折合分算法**：`折合分 = raw / raw_full × zhe_full`。达成度 = 折合总和 / 折合满分总和
7. **docx 模板**：用教务处官方空白模板（附件7），路径 `media/附件7.*.docx`
8. **分析文本必须数据驱动、内容详实**（铁律·2026-06-30 审计整改）：
   - **一、终结性考核分析**：必须包含（一）概况（分布特征判断"两极分化/相对集中"）、（二）分段百分比（7段均列出）、（三）典型错误+低分学生名单（≤5人写全名，>5人列举前5+等N人）
   - **二、教学目标达成分析**：必须包含目标排名、最弱考核环节名称+达成率、分班对比（如多班）
   - **三、存在问题**：学生≥4点（基础/综合/自主学习/平时vs期末），教师≥4点（专项训练/帮扶/资源/班级均衡），每点用数据支撑
   - **四、改进意见**：≥5条，每条指向前文发现的具体薄弱环节
   - 分析文本总字数≥1000字，不得使用纯模板占位文本
