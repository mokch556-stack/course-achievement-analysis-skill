## 🧠 自主智能规则（铁律，不可跳过）

### 规则0：折合权重约束 — 逐列SUM == 总评（P0，必过）

**问题**：同一考核环节（线上学习、课堂表现等）出现在多个教学目标中时，zhe_full 若不同目标重复使用同一权重，逐列求和会远超总评。

**检测规则**：
1. 从开课说明/教学大纲提取课程实际评分公式（如 `总评=平时×0.7+项目×0.3`）
2. 对每个考核环节，计算其**对总评的实际贡献权重**（合计=100）
3. 若该环节出现在 N 个目标中，则每个目标列的 zhe_full = 原始权重 ÷ N
4. **强制验证**：对任一学生，逐列折合求和 == 该生总评成绩（误差≤1分）

**脚本**：`scripts/verify_zhe_full.py` — 取前3名学生验算，PASS才继续。

### 规则1：xlsx 文件必须遍历全部 Sheet
- `*平时成绩报告单*.xlsx` 等文件可能有多个 Sheet，必须遍历 `wb.sheetnames`
- 跳过明显非数据Sheet（名称为纯数字的汇总Sheet），其余全读

### 规则2：子项→目标映射必须从教学大纲/开课说明提取
- 进入课程档案父目录，搜索 `*教学大纲*.docx` 和 `*开课说明*.docx`
- pandoc 提取文本，读取考核评价方式、项目→目标对应
- **阶段测试按项目归属**：项目1→目标1，项目2→目标2
- **覆盖性考核（线上/课堂/手册/期末）**：均匀分配到所有目标
- 找不到大纲/开课说明才问用户

### 规则3：按专业分表
- 一个专业一张达成度计算表 + 一份附件7报告
- 从班级名提取（如"互金251"→"互联网金融"）

### 规则4：阶段测试保留原始100分制
- 不提前折合、不取平均，在达成度计算时各自折合

### 规则5：数据归集完整性校验
- config 生成后立即交叉比对：系统成绩单 N 人，config M 人，必须 N==M
- 缺人时自动从成绩单补录（平时成绩齐全则补全，否则标记需人工处理）

## 🔥 docx 模板填充铁律

### 铁律 A：不删行、不增行、原位覆写
- **绝对禁止** `add_row()` / `_tbl.remove()` — 必破坏 vMerge 结构，导致大面积空白
- 所有数据**直接在模板原行上覆盖文本**，保留原行的合并单元格设定

### 铁律 B：写前先 dump，不凭假设
- 填充前必须 dump 模板的每行每列：text + vMerge + hMerge(gridSpan)
- 识别：哪些列合并（C6/C7分值A=横向合并）、哪些行同组（C9纵向合并）、分数段重复列
- 用 `scripts/probe_template.py` 自动 dump

### 铁律 C：合并单元格只在 vm_restart 行写
- vMerge=continue 的 cell 共享 restart 的 `_tc`，写入会覆盖
- 每个合并组只在首行（vm='restart'）写一次，其余行跳过

### 铁律 D：Table0 自适应裁剪
1. **删第二个班级区域**：单班时删掉模板中第二个班级的所有行（如 R13-R17）
2. **删重复分数段列**：检测相邻两列分数段名称相同→只保留一列
3. **删项目作业统计行**：课程无项目作业时删相关统计行

### 铁律 E：Table1 自适应裁剪
1. **删多余项目作业行**：模板每目标6行（5项+项目作业），实际5项时**直接删第6行**，不要打"—"占位
2. 目标数≠模板时，多余的整个目标区全删

### 铁律 F：多 run 文字替换
- 模板中单段文字可能被拆成多个 run（如"任课教师"、"： "、"刘娓娓" → 3个run）
- 必须用 `replace_in_cell_fuzzy()` — 逐 run 搜索 fragment 匹配，而非 `replace_in_cell()` 整段匹配
- **写新文本前先 `cell.paragraphs[0].clear()` 全部清空**，再写一个干净 run，避免原模板多run残留

### 铁律 G：课程性质不擅加
- config.course_nature 写什么就填什么，不加括号备注
- "必修课"就是"必修课"，不要写成"必修课（通识教育课）"

## 工作流（5步）

```
Step 0: 探查+归集 → 遍历Sheet + 读教学大纲 + aggregate_scores.py
Step 1: 构建 config → 按专业拆分 + 折合权重约束验证
Step 2: 生成 xlsx  → build_xlsx_school.py
Step 3: 生成 docx  → build_docx_newmedia.py（填充新媒体营销模板）
Step 4: 质量验证  → verify_zhe_full.py + 归集交叉比对
```

### Step 0: 探查目录 + 数据归集

1. 扫描课程档案目录，列出所有 `.xlsx` + Sheet名+列名
2. 向上2级目录搜索 `*教学大纲*.docx` 和 `*开课说明*.docx`
3. pandoc 提取 → 考核方式、评分权重、项目→目标映射
4. 运行 `scripts/aggregate_scores.py` 归集所有学生成绩

### Step 1: 构建 config.json，含折合约束验证

**关键：zhe_full 必须满足逐列SUM == 总评**

确定 zhe_full 的流程：
1. 从开课说明提取评分公式（如 `总评=平时50%+期末50%`）
2. 将百分制得分按评分公式换算为对总评的实际贡献
3. 同一环节在N个目标中出现 → zhe_full ÷ N
4. 取前3名学生验算：`sum(折合值) ≈ luru_zp`（误差≤1）

**config 结构**（与 v4 相同，增加 `course_nature`）：
```json
{
    "course_name": "大学计算机基础",
    "course_code": "3300000005",
    "term": "2025-2026学年第2学期",
    "course_nature": "必修课",
    "assessment": "考查",
    "major": "互联网金融",
    "classes": ["互金251"],
    "teacher": "邹芳",
    "hours": 32,
    "credits": 2,
    "targets": […],
    "students": […]
}
```

### Step 2: 生成 xlsx

```bash
python scripts/build_xlsx_school.py config_{专业名}.json output.xlsx
```

单Sheet学校标准宽表格式，折合得分 = 20列SUM公式（满分≤100）。

### Step 3: 生成 docx（填充新媒体营销模板）

```bash
python scripts/build_docx_newmedia.py config_{专业名}.json output.docx
```

**模板**：`《新媒体营销》试卷分析&教学目标达成情况-2025-2026-2_最终.docx`

**流程**：
1. 复制模板 → 按铁律 D/E 自适应裁剪行列
2. Table0 填充：课程信息行 + 成绩分段（按铁律 F 多run替换）
3. Table1 填充：4×5=20行数据（按铁律 C 只在vm_restart写合并列）
4. 分析文本生成：四段式，≥1000字数据驱动
5. 签名行保留模板原样

### Step 4: 质量验证

1. **折合约束**：`verify_zhe_full.py` → 取前3名学生，逐列SUM ≈ 总评
2. **归集完整性**：config学生数 vs 成绩单学生数，打印差异列表
3. **达成度一致性**：xlsx 和 docx 的达成度来自同一 config，数学基础统一

## 分析文本铁律（≥1000字，四段数据驱动）

### 一、终结性考核典型性错误及试题难度分析
- （一）概况：分布特征（"两极分化"/"相对集中"），平均分+及格率
- （二）分段百分比：7段全列出，标出峰值区间
- 50-59分段若≥30%，必须标注"可挽救区间"并给出具体人数
- （三）典型错误：归因到具体层面（基础概念→操作技能→综合应用），列不及格学生名单（≤5全名，>5前5+等N人）

### 二、课程教学目标达成分析
- 4目标达成度横向差异（最大差）
- 最弱考核环节名称+达成率
- **纯单班报告不写合班对比**（报告主体只有一个专业）

### 三、存在问题
- 学生≥4点：课堂投入、课后巩固、极端个例、班风
- 教师≥4点：预警失效、互动不均、练习反馈缺失、帮扶缺位
- 每点有具体数据支撑（"8人50-59分"而非"部分学生"）

### 四、改进意见
- ≥5条可落地措施，每条对应前文发现的薄弱环节
- 包含时间节点（"阶段测试后48小时内""从第一周起"）
- 极端个例必须有单独处理建议

## 注意事项

1. **折合约束是第一道关**：config阶段不过，后面全白做
2. **归集完立即交叉比对**：学生数不一致马上排查
3. **docx模板先dump再填**：不凭假设，不动行列结构
4. **config和xlsx必须同步**：zhe_full以xlsx为准，生成docx前从xlsx读回校验
5. **分析文本因课施策**：不同课程的问题点不同，不套用模板段落
6. **合班报告独立写**：每份报告只说自己的数据，不提合班的另一个专业
7. **终结性考核=期末考试成绩≠总评**：Table0成绩分段用期末成绩
