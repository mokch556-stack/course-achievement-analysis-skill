# 课程教学目标达成度分析技能

## 什么时候用

当用户需要：
- 为课程计算各教学目标达成度
- 生成教务处模板格式的**达成度计算表**（.xlsx），含Sheet1数据表、Sheet2映射关系、Sheet3示例学生
- 生成**附件7：课程教学目标达成情况分析报告**（.docx），含4段式结构
- 对应附件6清单要求：终结性考核典型性错误及试题难度分析、课程教学目标达成分析、存在问题、改进意见

**不要用**：
- 只需录入成绩（用成绩录入技能）
- 只需听课评价（用听课评价技能）
- 只需创建成果申报表（用 create_achievement_form 技能）

## ⚠️ 【强制】xlsx 完成后的完整结构（缺一不可）

整个完成态的xlsx = Sheet1 + Sheet2 + Sheet3。每个Sheet的每一行都有固定职责。

### Sheet1：达成度计算表（完整填写后）

```
┌─────────────────────────────────────────────────────────┐
│ R1:  课程名称 / 学期 / 专业 / 班级（合并所有列）       │ ← 行1 标题
├─────────────────────────────────────────────────────────┤
│ R2-R3: 课程信息（学分/学时/教师/代码等）              │ ← 行2-3 课程信息
├─────────────────────────────────────────────────────────┤
│ R4:  毕业要求指标点（每个目标合并对应列）               │ ← 行4 毕业要求
├─────────────────────────────────────────────────────────┤
│ R5:  课程教学目标（每个目标名称跨对应列）               │ ← 行5 教学目标
├─────────────────────────────────────────────────────────┤
│ R6:  考核环节（每个目标下分：平时/期末/...）           │ ← 行6 考核环节
├─────────────────────────────────────────────────────────┤
│ R7:  满分值（每列对应考核环节的满分分值）               │ ← 行7 满分值
├─────────────────────────────────────────────────────────┤
│ R8+: 学生数据（每人一行）                               │ ← 行8起 数据
│       序号│学号│姓名│目标1-平时│目标1-期末│...│折合总分│
├─────────────────────────────────────────────────────────┤
│ 平均分行:  各列平均值（所有数据列都要算）               │ ← **强制！不可缺**
├─────────────────────────────────────────────────────────┤
│ 各目标达成度行:  各目标合并对应列，写达成度值           │ ← **强制！合并+写值**
├─────────────────────────────────────────────────────────┤
│ 课程达成度行:  全部数据列合并为一个单元格，写总达成度   │ ← **强制！必须合并**
├─────────────────────────────────────────────────────────┤
│ 分班达成度行1:  电商231班各目标达成度                   │ ← **强制！分班汇报**
├─────────────────────────────────────────────────────────┤
│ 分班达成度行2:  电商232班各目标达成度                   │ ← **强制！分班汇报**
└─────────────────────────────────────────────────────────┘
```

**关键强制规则：**

| 强制项 | 具体要求 | 违规后果 |
|--------|---------|---------|
| **平均分行** | 所有数值列（平时/子项/期末/折合分）都要算平均值，保留2位小数 | 数据不完整 |
| **各目标达成度行** | 每个目标合并该目标对应的所有列 → 写达成度值（0.76格式，2位小数） | 格式不合规 |
| **课程达成度行** | 全部数据列合并为一个单元格 → 写课程整体达成度（如0.76） | 格式不合规 |
| **分班达成度行** | 先汇总231班所有学生各目标得分→算231班各目标达成度，再算232班 | 汇报不全 |

**分班达成度的计算方法：**
```
1. 筛选出电商231班的所有学生
2. 对该班每个目标：全班该目标折合分之和 / 全班该目标满分之和 = 该班该目标达成度
3. 同样方法算232班
4. 写成两行（如"电商231班达成度：目标1=0.78, 目标2=0.75..."）
```

### Sheet2：满分来源详解（映射关系表）

```
考核项目       满分   目标1  目标2  ...  目标K   说明
──────────────────────────────────────────────
课堂表现-项目一  30    13.44   0      0     0    按课点归属
学习通           20    7.14   3.57  3.57  5.72  按课点比例
课程竞赛         20    7.14   3.57  3.57  5.72  按课点比例
课堂笔记         20    7.14   3.57  3.57  5.72  按课点比例（需补偿调整）
实训报告         30    10.71  5.36  5.36  8.57  按课点比例
期末项目作业    100    40     10     20    30    按评分标准4维度
──────────────────────────────────────────────
合计            100    xx     xx    xx    xx    各目标满分
```

**满分矩阵计算规则**（强制步骤）：
1. **课堂表现（阶段性测试）**：按项目内容归属到对应目标（项目一→目标1等）
2. **通识子项（学习通/课程竞赛/课堂笔记/实训报告）**：按**课点比例**分配到各目标
   - 各目标课点数 = 教学大纲表格②
   - 分配比例 = 该目标课点数 / 总课点数
3. **期末项目作业**：按评分标准4维度直接对应（40/10/20/30）
4. **笔记列需补偿调整**：以上分配后各目标合计与教学大纲规定比例不一致时 → 微调笔记列
5. **验证**：各目标合计横向加总 = 100 ✓

### Sheet3：示例：{学生姓名}

- 选1个成绩中等的学生（约75分）
- 复制Sheet1该生的数据行
- 在旁边加**列注释**：
  - `=D8*满分矩阵` → 拆解到各目标折合分
  - `=折合分/目标满分` → 该目标达成度
  - `=SUM(折合分)` → 总折合分
  - `=总折合分/100` → 课程达成度
- 作用是让用户一眼看明白计算公式

def find_text_run(p):
    """在段落中查找含有非空<w:t>的run（不一定在第一个run）"""
    for run in p.findall(f'{W}r'):

### 第三步：核验（三核查）+ 整理 + 打开

#### 3a：数据核验（三核查必做，不可跳过）

| 维度 | 检查项 | 通过条件 | 验证方法 |
|------|--------|---------|---------|
| 数据一致性 | ①成绩分布（各分数段人数） ②最高/最低分+姓名 ③平均分 ④及格率 ⑤不及格学生名单+分数 | 与成绩登记表原始数据 100% 一致 | Python脚本逐条对照 |
| 折合分对齐 | ⑥每名学生的折合总分=录入系统总评（误差≤0.05） | 100%通过 | 计算 sum(goal_zhe) 对比录入总评 |
| 平时子项对齐 | ⑦报告单子项和与录入平时分一致（不一致时用**余数分配法**对齐） | 对齐后子项和=录入平时分 | 先检验s=|xx+kt+jd+sc - luru_ps|，若≥0.5则执行余数分配法 |
| 子项完整性 | ⑧平时各子项折合+期末各目标折合=录入总评 | 100%通过 | 逐学生验证 sum(goal_zhe[gi]) = zp |
| 结构完整性 | ①平均分行存在 ②各目标达成度行存在+合并 ③课程达成度行存在+合并 ④分班达成度2行存在 | 全部存在且值正确 | 直接读xlsx验证 |
| 教学大纲一致性 | ①教学目标数量+名称 ②学分 ③考核方式 ④课点归属 | 与教学大纲.docx一致 | 读取文件对照 |
| 开课说明一致性 | ①学期 ②考核构成 ③成绩比例 ④二级矩阵 | 与开课说明.docx一致 | 读取文件对照 |

**⚠️ 成绩登记表列结构关键**：
- col 4 = 平时成绩，col 5 = 期末成绩（不是总评col 7！）
- 两个班列结构**完全相同**（先探查表头行再确认）
- ⚠️ 成绩分布必须用期末成绩col 5（不是总评，也不是平时）

**⚠️ 常见错误（已有血泪教训）**：
| 错误 | 后果 | 预防 |

## 特殊场景处理

### 平时子项与录入不一致时的对齐策略（余数分配法）

当学生平时成绩报告单的子项和（xx+kt+jd+sc）与录入系统的平时分（luru_ps）不一致时：

**原因**：部分课程的成绩报告单可能存在抄录误差，但录入系统的总评是权威的。

**对齐原则**：子项总和必须等于录入平时分。

**算法步骤**：
1. 计算比例因子：r = luru_ps / (xx+kt+jd+sc)
2. 按比例缩放每项：items = [xx*r, kt*r, jd*r, sc*r]
3. 取整数部分：fl = [int(v) for v in items]
4. 计算余数：left = round(luru_ps - sum(fl))
5. 按小数部分从大到小排序，前 left 项各+1

**Python代码**：
```python
def align_subs(xx, kt, jd, sc, luru_ps):
    s = xx+kt+jd+sc
    if abs(s - luru_ps) < 0.5:
        return int(round(xx)), int(round(kt)), int(round(jd)), int(round(sc))
    r = luru_ps / s
    items = [xx*r, kt*r, jd*r, sc*r]
    fl = [int(v) for v in items]
    left = int(round(luru_ps - sum(fl)))
    dec = sorted(enumerate(v-int(v) for v in items), key=lambda x: x[1], reverse=True)
    for i in range(left):
        fl[dec[i][0]] += 1
    return fl[0], fl[1], fl[2], fl[3]
```

#### 折合总分计算逻辑

```
折合总分 ≠ Σ(子项)×比例  ← 不重新计算
折合总分 = 录入总评列        ← 直接取值
```

**为什么**：录入系统已经有最终总评（可能涉及四舍五入/手动调整），再算一遍必然会不一致。只要最后一名学生的折合总分等于录入总评，就是正确的。

**关键**：分析行必须在Table1重建时**保留**（不删除、不重建），重追加回表格末尾。这样才能保留模板原有的BOLD标题和占位符。重追加后row index发生变化，分析行在15行表格中的index=12（R0=header, R1-R11=data, R12=analysis, R13=sig, R14=blank）。

### python-docx cell.merge() 陷阱（2026-06-28 v7 发现）

**现象**：连续对同一表的不同列调用 `cell.merge()` 时，第二次调用报错：
```
no tc element at grid_offset=N
```
或 `raise_on_inverted_L` 异常。

**根因**：`cell.merge()` 内部调用 `_span_dimensions()` → 依赖表格的 grid 偏移计算。第一次 merge 后 grid 被修改（gridSpan 变化、vMerge 属性添加），导致后续 `cells[ci]` 的 grid_offset 与 XML 实际位置不一致。

**解决方案（推荐）**：用 XML 直接设 vMerge，不依赖 python-docx 的 merge()：

```python
from lxml import etree
WML = 'http://schemas.openxmlformats.org/wordprocessingml/2006/main'

def set_vmerge(cell, val=None):
    """val='restart' 为首行, val=None/'continue' 为续行"""
    tc = cell._tc
    tcPr = tc.find(f'{{{WML}}}tcPr')
    if tcPr is None:
        tcPr = etree.SubElement(tc, f'{{{WML}}}tcPr')
    vMerge = etree.SubElement(tcPr, f'{{{WML}}}vMerge')
    if val == 'restart':
        vMerge.set(f'{{{WML}}}val', 'restart')
    # continue: 不设val属性

# 用法：垂直合并R1-R5的C1列
set_vmerge(t1.rows[1].cells[1], 'restart')  # 首行
for r in range(2, 6):
    set_vmerge(t1.rows[r].cells[1])          # 续行
```

**备选方案**：每次 merge() 后 `doc.save()` + `doc = Document()` 重新加载（慢，仅适用于操作次数少的情况）。

### Table1 合并 + 删除汇总行完整攻略（v7血训）

> **场景**：附件7 Table1需要：C1毕业要求/C2指标点/C3-C4课程教学目标都做竖向合并（R1-R5目标1，R6-R10目标2），同时删除R11课程达成度汇总行。

**三个关键教训**：

1. **删行前先合并**：删除R11会改变后续行的引用索引，合并操作必须在对R11操作之前完成。

2. **C3-C4是横向合并，不能删tc4**：C3和C4在模板中就是一个合并单元格（gridSpan=2的tc3 + vMerge continuation的tc4）。修复时只需设置vMerge属性，**不要删除tc4元素**，否则grid列总数从10变成9，破坏表格结构。

3. **使用XML层全套操作**，避开python-docx的cell访问（merge后grid重算导致`cells[i]`不可靠）：

```python
from lxml import etree
WML = 'http://schemas.openxmlformats.org/wordprocessingml/2006/main'

# 1. 删除R11(课程达成度行)
tbl.remove(trs[11])

# 2. C1-C4竖向合并（XML层操作）
for goal_range in [(1, 6), (6, 11)]:  # R1-R5, R6-R10
    for ri in range(goal_range[0], goal_range[1]):
        tcs = trs[ri].findall(f'{{{WML}}}tc')
        for ci in [1, 2, 3, 4]:  # C1-C4都竖向合并
            tc = tcs[ci]
            tcPr = tc.find(f'{{{WML}}}tcPr')
            if tcPr is None:
                tcPr = etree.Element(f'{{{WML}}}tcPr')
                tc.insert(0, tcPr)
            for vm in tcPr.findall(f'{{{WML}}}vMerge'):
                tcPr.remove(vm)
            vm = etree.SubElement(tcPr, f'{{{WML}}}vMerge')
            if ri == goal_range[0]:
                vm.set(f'{{{WML}}}val', 'restart')
            # else: continue (no val attribute)
            # 续行清空文本
            if ri != goal_range[0]:
                for p in tc.findall(f'.//{{{WML}}}p'):
                    for r in p.findall(f'{{{WML}}}r'):
                        for t in r.findall(f'{{{WML}}}t'):
                            t.text = ''
```

4. **分析文本重写**：用`p.clear()`清空段落 + `p.add_run(text)`重建，避免`p.runs[0].text =`方式在复杂run结构下失效。

### C5考核环节列：必须逐行列出，不能竖向合并
